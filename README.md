# 머신러닝을 활용한 배틀그라운드 악성유저 데이터 기반 탐지

## 프로젝트 개요 
- PUBG_API를 활용하여 정상, 비정상 유저의 데이터를 추출, 변환, 저장하는 과정을 거쳐 데이터를 수집
- 총 110만건의 게임 데이터 수집 및 분석 
- XGBoost 기반 이진 분류 모델 (정상, 악성)
- 높은 재현율(99.5%) 악성유저 99.5% 탐지 

**기간** 24/09 ~ 24/11 | **기술**: Python, PySpark, XGBoost, Scikit-learn

##  모델 성능

| 지표 | XGBoost | Random Forest |
|------|---------|---------------|
| **Accuracy** | **97.3%** ↑ | 93.1% |
| **Precision** | **99.5%** | 91.2% |
| **Recall** | **95.4%** ↑ | 95.6% |
| **F1-Score** | **97.4%** ↑ | 93.3% |
| **ROC-AUC** | **0.996** ↑ | 0.981 |

## 핵심기술 

### 1: 데이터 수집과 전처리 
PUBG API를 통한 3단계 데이터 수집 
1. 리더보드 -> 플레이어ID 추출 
2. 플레이어별 매치 ID를 수집 
3. 매치 상세 데이터 수집 

필터링 기준 
* 피해량 > 0 
* 생존시간 >= 240s
-> 두 기준은 게임이 시작하고 곧바로 죽었거나, 아무 활동을 하지않은 유저를 제외하여
   유의미한 데이터가 추출될 수 있도록 필터링  

오버샘플링 + SMOTE 
* 정상 데이터: 100만건, 악성 데이터: 5천건 
* SMOTE로 소수 클래스 오버 샘플링을 진행, 훈련 데이터만 적용하여 수행 


## 통계적 검증

정상 vs 악성 유저의 주요 특성 차이 (T-test):

| 특성 | 정상 평균 | 악성 평균 | P-value |
|------|----------|----------|---------|
| Kill | 2.14 | 5.82 | < 0.001 |
| Headshot | 0.21 | 1.23 | < 0.001 |
| Damage | 187.2 | 542.8 | < 0.001 |
| Win Rate | 0.042 | 0.156 | < 0.001 |

* 모든 특성에서 통계적으로 유의미한 차이

## 하이퍼 파라미터 최적화 
RandomizedSeachCV 적용:
* 탐색 공간: 324개 조합, 샘플링: 50개 조합 무작위 선택 
* 교차 검증: 3-Fold
최적 파라미터 -> 
n_estimators=100, max_depth=5, learning_rate=0.1,
subsample=0.8, colsample_bytree=0.8, gamma=0.1

##  데이터 파이프라인

PUBG API 데이터 수집 (pubg_pipeline.py)
↓
결측값 제거 & 이상치 필터링 (Jupyter Notebook)
↓
특성 엔지니어링 (역수 변환)
↓

↓
SMOTE + 과대 샘플링
↓
XGBoost 훈련 및 평가 (xgb_modeling.py)
↓
모델 저장 (.pkl)


1. PUBG_데이터추출.ipynb , pubg_pipeline.py 
PUBG_데이터분석 노트북 파일은 프로젝트 초기 따로따로 수집한 코드입니다.  
해당 파일로 모든 데이터 수집, 변환, 적재가 가능하지만 약간 불편함이 있어. 
pubg_pipeline.py파일로 파이프라인을 구축하여 만들었습니다. 

2. PUBG_데이터분석.ipynb

결과는 모두 나와있지만, 난잡하여 Notion으로 정리한 내용을 보는걸 추천드립니다.
